# CareVault

Tauri v2 + React + TypeScript personal health tracking app.

CareVault is a local-first health notebook for manually tracking:

- blood pressure and blood glucose readings with non-positive/blank draft validation and saved-vital panel summary chips for type/status mix
- autosave-preserved feedback when new vitals, visits, symptoms, questions, labs, and document notes are added
- profile number validation that blocks direct or imported non-positive age, height, weight, and waist circumference values and avoids dangling units for blank profile fields
- profile waist input helper that switches between Korean male, female, and unspecified waist-circumference standards
- lab results with user-entered reference ranges, saved-lab panel summary chips, common CBC (WBC/RBC/Hct/PLT)/diabetes/lipid/liver-function presets, and preset-derived official-source recovery for older or direct-entered lab rows such as WBC
- Korean adult BMI and waist-circumference screening from age/sex/height/weight/waist profile data
- Korean standard sex-applicability summary chips, current-profile sex standard notes, and quick filters that distinguish male/female-specific thresholds from adult common BMI, blood-pressure, glucose, low-glucose, marked-hyperglycemia, A1C/FPG/PP2, BUN/Cr/eGFR/UACR kidney-function, Na/K electrolyte, common lipid, AST/ALT liver, Alb/TP protein/nutrition, Ca calcium, P phosphate, UA uric-acid metabolic helper, and CBC WBC/RBC/Hct/PLT helper rules
- one-click copy for Korean health-standard coverage, dashboard BMI/waist/BP/glucose metric-standard source labels, current-profile sex notes, source labels, and the currently selected standards quick-filter category across BMI/waist/blood-pressure/glucose/low-glucose/marked-hyperglycemia/A1C/BUN/Cr/eGFR/UACR/Na/K/lipid/HDL/GGT/AST/ALT/Alb/Ca/P/UA/Hgb/WBC/RBC/Hct/ANC/PLT numeric and risk-threshold ranges plus cancer-patient fever/infection contact thresholds, with full-copy behavior preserved when `전체` is selected, plus fever/infection standards-card shortcuts that prepare source-retaining symptom record and clinician-question drafts
- cervical-cancer care notes with top summary chips, source-backed warning-record field cards for when/what/how-much/with-what symptoms to record, priority record/question/recovery checklists, warning-sign logging, profile-based national screening quick-checks with 산정특례기간-based deferral checks, screening-versus-diagnostic-test distinction prompts, suspected-symptom diagnostic-test checklists, source-backed stage-explanation notes for clinic conversations, one-click screening-to-question, structured warning-to-symptom recording drafts that reuse the same warning-record field guide, and checklist-to-record drafts with source-backed care-queue/export evidence, source-URL-retaining symptom and follow-up question drafts, treatment-selection consultation prompts, 골반 방사선치료 후 폐경/난소기능 consultation prompts, 장·방광 후기 변화 contact-standard question drafts, recovery timing checks, recurrence/follow-up interval memos, post-treatment sex-life pain/dryness consultation memos, pregnancy/birth planning consultation memos, food/supplement consultation memos, urinary/bowel/bleeding change memos, late 장폐색·혈변·혈뇨 contact-standard memos, lymphedema infection/worsening-signal memos, follow-up schedule/result checks, national screening result/cost notes, HPV vaccine versus screening-continuation notes and question drafts, separate HPV vaccination schedule/observation plus delayed-dose/no-booster memos, smoking/sexual-history risk-factor consultation memos, lymphedema exercise notes, national screening/prevention memos, and official Korean source links

without launching Tauri.

## Storage Notes

The app now uses SQLite when it runs inside Tauri and falls back to localStorage in a browser preview. If browser localStorage is blocked or quota-limited, the app degrades to temporary memory instead of breaking hydration or save feedback, and the save status labels state that explicitly as `임시 메모리에만 저장됨` or `임시 메모리에만 자동 저장됨`. Document selection uses Tauri dialog `fileAccessMode: "copy"` in desktop runtime so the selected file is copied into the app sandbox and the copied path is stored as document metadata.
